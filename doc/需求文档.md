skillsops

需求背景
团队内部积累了大量可复用的 Skills（自动化脚本、工作流模板、效率工具等），目前散落在各人电脑和不同仓库中：
- 不知道团队里有哪些现成的 Skill 可以用，重复造轮子
- Skill 质量参差不齐，没有评价和筛选机制
- 作者更新了版本，使用者不知道，还在用旧版
- 缺乏使用数据，不清楚哪些 Skill 真正有价值，无法指导后续投入
需要一个 Skills 运营平台，支持发布、发现、安装和评价团队内部的 Skills。
产品方案
角色定义
- 管理员：审核 Skill 上架、管理分类、查看全局运营数据
- 普通用户：发布自己的 Skill、浏览和安装他人的 Skill、提交评分和评价
页面规划
登录/注册页：用户认证入口。
Skill 市场页：展示已上架的 Skill 列表，每项显示名称、描述、分类标签、作者、当前版本号、安装量、平均评分。支持按分类、关键词、排序方式（热门/最新/评分）筛选。
Skill 详情页：查看 Skill 完整信息、版本历史列表、用户评价列表。可安装、提交评分和评价。管理员可审核上架/下架。
个人工作台页：作为作者——我发布的 Skill 列表、创建新 Skill、发布新版本、编辑信息。作为用户——我安装的 Skill 列表。管理员额外展示待审核列表和运营统计。
核心业务流程
作者创建 Skill 并提交上架申请
  → 管理员审核（通过 → 上架 / 拒绝 → 退回修改）
  → 用户浏览市场，安装 Skill
  → 用户对已安装的 Skill 评分和评价
  → 作者发布新版本（已上架的 Skill 直接更新，无需重新审核）

业务规则
- Skill 有生命周期：草稿 → 审核中 → 已上架 → 已下架
- 每个 Skill 支持多版本发布，版本号由作者指定
- 用户安装 Skill 时默认安装最新版本，安装记录关联具体版本
- 每个用户对同一 Skill 只能提交一次评分（1-5 分），可修改
- 已下架的 Skill 不再出现在市场列表中，但已安装的用户仍可查看
- 只有作者本人和管理员可以编辑 Skill 信息
功能清单
用户认证
- 注册、登录、退出
- 基于 Token 的接口认证
- 无需考虑接入公司 OpenID 登录
Skill 发布与管理
- 创建 Skill：名称、描述、分类、仓库地址/文档链接
- 提交上架申请
- 发布新版本：版本号、更新说明
- 编辑 Skill 信息（仅作者和管理员）
- 查看我发布的 Skill 列表
Skill 市场
- Skill 列表：支持按分类、关键词搜索、排序方式（安装量/评分/最新发布）筛选，支持分页
- Skill 详情：完整信息、版本历史列表、用户评价列表
- 安装 Skill（记录安装人和版本，更新安装计数）
- 查看我已安装的 Skill 列表
评价体系
- 对已安装的 Skill 评分（1-5 分）和评价
- 每人每个 Skill 只能评一次，允许修改
- Skill 的平均评分实时展示
运营管理（管理员）
- 审核上架申请（通过/拒绝，附理由）
- 下架违规 Skill
- 管理分类标签
- 查看运营统计（Skill 总数、安装趋势、活跃作者、热门 Skill 排行）
本题功能模块较多，如时间紧张可优先保证市场浏览、安装、发布核心链路，评价体系和运营统计作为次优先级。
